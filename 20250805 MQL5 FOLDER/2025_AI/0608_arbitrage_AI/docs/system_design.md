### Project File Structure

This is the recommended file structure for our fully functional and refactored arbitrage system. It organizes the code logically, separating the core logic, configuration, MQL5 code, and results.

```
Triangular-Arbitrage-Bot/
│
├── forex_arbitrage_system.py      # <-- The LIVE Python trading bot (High-frequency loop)
├── arbitrage_backtester.py        # <-- The Python backtesting engine (Historical simulation)
│
├── arbitrage_config.ini           # <-- Main configuration for Python scripts (MT5 path, etc.)
├── requirements.txt               # <-- Python package dependencies (metatrader5, pandas, etc.)
├── setup.py                       # <-- Simple setup script to install dependencies
│
├── mql5_expert_advisor/
│   └── ArbitrageTrader.mq5        # <-- The refactored, standalone MQL5 Expert Advisor
│
└── results/
    ├── charts/
    │   └── backtest_results.png   # <-- Output equity curve chart from the backtester
    └── logs/
        └── arbitrage_system.log   # <-- Log file generated by the live system

```

### Logical Design and Data Flow

Here is a visualization of how the system processes information, from initialization to trade execution.

#### 1. Live System Logic (`forex_arbitrage_system.py`)

This flowchart illustrates the high-frequency loop of the live trading bot.

```
[ START ]
    │
    V
[ Initialize System ]
    │   1. Read arbitrage_config.ini
    │   2. Connect to MetaTrader 5 Terminal
    │   3. Programmatically generate all Arbitrage Triangles
    │   4. Subscribe to market data for all required currency pairs
    │
    V
┌───────────────────────────────────────────────────┐
│     --- HIGH-FREQUENCY LOOP (Runs continuously) ---     │
│                                                   │
│   For each Arbitrage Triangle in the list:        │
│   ┌───────────────────────────────────────────────┤
│   │                                               │
│   │  1. Fetch LATEST Ticks (2 base pairs)         │
│   │     (e.g., Get EURUSD & GBPUSD ticks)         │
│   │                                               │
│   │  2. Calculate Synthetic Price                 │
│   │     (e.g., Synthetic EURGBP = EURUSD / GBPUSD)│
│   │                                               │
│   │  3. Fetch REAL Market Price (cross pair)      │
│   │     (e.g., Get the real EURGBP price)         │
│   │                                               │
│   │  (DECISION)                                   │
│   │  Is │Real Price - Synthetic Price│ > Min Spread?
│   │       │                                       │
│   │       └─(No)─> Continue to next triangle      │
│   │       │                                       │
│   │      (Yes)                                    │
│   │       │                                       │
│   │       V                                       │
│   │  [ ARBITRAGE OPPORTUNITY FOUND ]              │
│   │       │                                       │
│   │       V                                       │
│   │  (RISK CHECKS)                                │
│   │    - Max open trades not reached?             │
│   │    - No existing position for this symbol?    │
│   │       │                                       │
│   │       └─(Fail)─> Log and skip                 │
│   │       │                                       │
│   │      (Pass)                                   │
│   │       │                                       │
│   │       V                                       │
│   │  [ EXECUTE TRADE ]                            │
│   │    - Send MARKET Order (BUY or SELL) on the   │
│   │      cross pair with TP/SL.                   │
│   │                                               │
│   └───────────────────────────────────────────────┘
│                                                   │
└───────────────────────────────────────────────────┘
    │
    V
[ System runs until manually stopped (Ctrl+C) ]
```

#### 2. Backtester Logic (`arbitrage_backtester.py`)

This flowchart shows how the backtester simulates the strategy over a historical period. The key difference is that it operates on a complete dataset rather than real-time ticks.

```
[ START ]
    │
    V
[ Initialize Backtest ]
    │   1. Read arbitrage_config.ini
    │   2. Connect to MT5 and generate triangles
    │   3. Fetch ALL historical M1 data for a given date range (e.g., 3 months)
    │      for every required symbol.
    │
    V
[ Prepare Data ]
    │   1. Combine all individual symbol histories into one large, time-aligned
    │      "Master DataFrame".
    │   2. Forward-fill any missing data points to create a continuous market view.
    │
    V
┌────────────────────────────────────────────────────────────┐
│  --- SIMULATION LOOP (Iterates minute-by-minute over data) ---   │
│                                                            │
│ For each minute (row) in the Master DataFrame:             │
│ ┌──────────────────────────────────────────────────────────┤
│ │                                                          │
│ │  1. Manage Open Positions                                │
│ │     - For each trade in the 'open_positions' list:       │
│ │       - Check if current price hits the Take Profit or Stop Loss.
│ │       - If yes: "Close" the trade, record the profit/loss, and
│ │         update the equity curve.                         │
│ │                                                          │
│ │  2. Look For New Trades                                  │
│ │     - Using prices from the CURRENT minute in the data:  │
│ │     - The logic is IDENTICAL to the live system loop ->  │
│ │       (Calculate Synthetic -> Compare to Real -> Check Spread)
│ │       │                                                  │
│ │       └─(No Opportunity)─> Continue to next check      │
│ │       │                                                  │
│ │      (Opportunity Found)                               │
│ │       │                                                  │
│ │       V                                                  │
│ │  [ OPEN TRADE ]                                          │
│ │    - Add a new position entry to the 'open_positions'    │
│ │      list with its entry price, TP, and SL.              │
│ │                                                          │
│ └──────────────────────────────────────────────────────────┘
│                                                            │
└────────────────────────────────────────────────────────────┘
    │
    V
[ END OF DATA ]
    │
    V
[ Generate Results ]
    │   1. Calculate final performance metrics (Win Rate, Profit, Drawdown).
    │   2. Create and save the `backtest_results.png` equity curve chart.
    │
    V
[ FINISH ]
```
