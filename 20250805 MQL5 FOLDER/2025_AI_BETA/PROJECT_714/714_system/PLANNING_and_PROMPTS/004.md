**MQL5 Expert Advisor: Core 714 Method Implementation**

This EA will be designed for the M5 timeframe and will focus on the crucial time-based setup and initial trend observation described in the strategy. It will include visual indicators.

**Key Features to Implement:**

1.  **Input Parameters:** Allow the user to define:
    *   The specific UTC+2 target time (e.g., 13:00).
    *   The Broker Server's GMT offset. (This is crucial for the EA to calculate when the target UTC+2 time occurs on the server).
    *   Observation window duration after the main time (e.g., 60 minutes, as discussed).
    *   Other standard EA inputs: Magic Number (to identify trades), lot size or risk percentage, Stop Loss distance (in pips or a multiple, potentially a placeholder for now), Take Profit target rules (placeholder), max spread, etc.
2.  **Time Zone Synchronization:**
    *   The EA needs to correctly determine the UTC+2 equivalent time on the broker's server using the server's current time and the GMT offset provided by the user or calculated internally.
    *   Verify the chart is on the M5 timeframe.
3.  **Visual Indicators (Vertical Lines):**
    *   Upon initialization or at the start of each trading day/relevant period, the EA will automatically draw vertical lines on the M5 chart at the specified key times (primarily 1 PM UTC+2, and potentially other significant times like 7 AM or 4 PM UTC+2/SAST as needed). These lines will serve as visual markers for the trader and the EA's logic.
    *   These lines should be persistent and redraw correctly on historical data if the chart is shifted.
4.  **Initial Trend Observation:**
    *   After the main target time (e.g., 1 PM UTC+2), the EA will monitor price action for a defined observation period (e.g., the next hour until 2 PM UTC+2).
    *   It will determine the dominant direction of the price move during this period (e.g., by comparing the closing price after the observation window to the price at the start time). This establishes the expected "trade against" direction.
5.  **Placeholder for Entry Logic:**
    *   The EA will identify that the initial observation period has ended and that the dominant trend for trading against has been determined.
    *   It will have a placeholder section (likely a separate function) where your advanced OB/FVG detection and entry confirmation logic will be integrated later. This part won't execute trades in this initial version, but it lays the groundwork.
6.  **Exit Logic:**
    *   The EA can include basic logic to close any open trades if the defined daily trading period or market hours are ending (e.g., by 22h00 UTC+2).

**What this initial version would NOT do (yet):**

*   **Identify specific entry *levels* (like precise OB, FVG, S/R zones) programmatically.** This requires integrating your specialized code.
*   **Execute trades based on complex confluence conditions:** The core entry *trigger* logic (price interacting with the identified levels) will be the part you add.

//| Check if it's a new day to reset variables (Optional but useful) |
//+------------------------------------------------------------------+
bool IsNewDay()
{
    static datetime last_day_check = 0;
    datetime now = TimeCurrent();
    
    // Get start of current day (server time)
    datetime today_start_server = now - now%86400;

    if(last_day_check == 0)
    {
        last_day_check = today_start_server;
        return false; // Not a new day yet on first run
    }
    
    if(today_start_server > last_day_check)
    {
        last_day_check = today_start_server;
        // --- Reset Daily Specifics for New Day ---
        g_InitialBias = 0;
        g_KeyPrice_At_KeyTime = 0;
        // Recalculate and redraw vertical lines for the new day
        OnDeinit(REASON_PARAMETERS); // Clean old lines
        OnInit(); // Re-initialize for the new day
        Print("--- New Day Detected ---");
        return true;
    }
    
    return false;
}
