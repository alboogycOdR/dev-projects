Yes, absolutely! Let's focus on fixing the Order Block (OB) detection scan. 

The primary issue seems to be how the scan range is defined and passed to `DetectOrderBlocksForToday`, and then how that function uses those indices to copy historical data.

**Key Problems to Address in OB Detection:**

1.  **Scan Range Definition:** 
When the OB scan is triggered (e.g., at 12:05 server time after bias is determined), the `scan_end_period_time` used in previous attempts (`g_ObservationEndTime_Server + PeriodSeconds(Period()) * ob_Lookback_Bars_For_Impulse * 2`) could point to a time far in the *future* relative to the current backtesting tick. 

`iBarShift` for a future time will not yield useful historical indices for scanning past OBs.


2.  **Indices for `DetectOrderBlocksForToday`:** 
This function expects `scan_newest_chart_index` to be the chart index of the most recent bar in your scan window, and `scan_oldest_chart_index` to be the chart index of the oldest bar in your scan window. Remember, chart indices go `0` (current forming bar), `1` (last closed bar), `2` (second last closed), and so on, increasing for older bars.


3.  **Data Copying in `DetectOrderBlocksForToday`:** 
The logic for `copy_start_chart_index` and `num_bars_to_copy` needs to be robust, ensuring it correctly maps the desired scan range (plus lookaheads for impulse checks) to the available historical bar data.

**Step 2: Make `DetectOrderBlocksForToday` More Robust with Copied Data Indexing**

The `DetectOrderBlocksForToday` function needs to correctly use the `scan_newest_chart_index` and `scan_oldest_chart_index` it receives.
It will loop from `chart_idx = scan_newest_chart_index` (the newer bar) up to `chart_idx = scan_oldest_chart_index` (the older bar).

**Step 3: Review `IsBullishOrderBlockCandidate` and `IsBearishOrderBlockCandidate`**

The core logic of these functions regarding impulse (cumulative move + consecutive candles) and the structure of the OB itself (no higher high / no lower low within `ob_MaxBlockCandles`) seems conceptually okay, but the array indexing for impulse needs to be carefully reviewed when using `ArraySetAsSeries(true)`.

If `ArraySetAsSeries(true)` is used for `open_arr`, `close_arr` etc., then:
*   `arr[0]` is the *last closed bar (shift 1 on chart)*.
*   `arr[1]` is the *second last closed bar (shift 2 on chart)*.
*   And so on.
*   An impulse *after* an OB candidate at `arr[arr_idx]` means checking bars `arr[arr_idx-1]`, `arr[arr_idx-2]`, ..., `arr[arr_idx - ob_Lookback_Bars_For_Impulse]`.

```mql5
// Example refinement for IsBullishOrderBlockCandidate when arrays are series:
bool IsBullishOrderBlockCandidate(int candidate_arr_idx, // This index IS the shift on chart when array is series
                                const double &open_arr[], const double &high_arr[], 
                                const double &low_arr[], const double &close_arr[], int array_length_copied)
{
   // OB Candidate candle (at candidate_arr_idx) must be bearish
   if (close_arr[candidate_arr_idx] >= open_arr[candidate_arr_idx]) return false;

   // Check there are enough *newer* bars for impulse check
   if (candidate_arr_idx < ob_Lookback_Bars_For_Impulse) return false; // Needs at least ob_Lookback_Bars_For_Impulse bars newer than it

   double cumulative_move_after_ob = 0;
   bool found_strong_move = false;

   // Scan *newer* bars (from candidate_arr_idx - 1 down to candidate_arr_idx - ob_Lookback_Bars_For_Impulse)
   for (int i = 1; i <= ob_Lookback_Bars_For_Impulse; i++) 
   {
      int impulse_check_arr_idx = candidate_arr_idx - i;
      if (impulse_check_arr_idx < 0) break; // Ran out of newer bars in array

      if (close_arr[impulse_check_arr_idx] > open_arr[impulse_check_arr_idx]) { // Bullish candle
         cumulative_move_after_ob += (close_arr[impulse_check_arr_idx] - open_arr[impulse_check_arr_idx]);
         if ((cumulative_move_after_ob / GetPipValue()) >= ob_MinMovePips) {
            bool consecutive_bullish_met = true;
            // Check consecutive from candle immediately after OB (candidate_arr_idx - 1) up to current impulse_check_arr_idx
            for(int k = 1; k <= i; k++) { 
               int consecutive_check_idx = candidate_arr_idx - k;
               if(close_arr[consecutive_check_idx] <= open_arr[consecutive_check_idx]) {
                  consecutive_bullish_met = false;
                  break;
               }
            }
            if(consecutive_bullish_met && i >= 2) { // Impulse move itself must be at least 2 bars
               found_strong_move = true;
               break; 
            }
         }
      } else {
         break; // Non-bullish candle breaks the impulse sequence
      }
   }
   if (!found_strong_move) return false;

   // Check structure of the OB block itself (candidate + ob_MaxBlockCandles-1 newer candles forming the block)
   // No high in the block period (candidate_arr_idx down to candidate_arr_idx - (ob_MaxBlockCandles-1))
   // should be higher than the candidate_arr_idx's high.
   // THIS LOGIC WAS for older candle forming OB and impulse being newer.
   // If the OB candidate IS the oldest candle of a potential block:
   // The current OB logic: candidate is one candle. Then checks next MaxBlockCandles-1 *after* it for structure.
```
