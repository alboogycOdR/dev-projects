    *   **The Fix:** I have simplified the reset logic significantly. Now, the `state` is only reset when a **new CRT range candle is found** via `f_getTimedRange()`. This is far more robust. For the 1-Hour model, a new candle is found each day. For the 4-Hour model, a new `8 AM (H4)` candle is only found once per day, giving the setup ample time to play out without being prematurely reset. This resolves the core issue of visual elements failing to render.

2.  **Crucial Input Settings Restored:**
    *   **`Advanced Contextual Filters`** group is back. You can now toggle `i_use_htf_kl_filter`, which is a critical component for screening for A+ quality setups.

    *   The indicator now uses two different internal timeframe variables, `CRT_TIMEFRAME` (H1 or H4) and `MSS_TIMEFRAME` (M15 or H1). The core functions (`f_getTimedRange`, `f_getLastSwing`) now correctly request data from the appropriate timeframe based on which mode you have selected. For example, if you choose "4-Hour CRT," it will get its range from the H4 chart but look for the MSS confirmation on the H1 chart, exactly as per the methodology.
