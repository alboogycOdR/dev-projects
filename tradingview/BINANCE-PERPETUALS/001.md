## **Project Plan & Product Requirements**

**Title:** Binance Perpetual Futures Trading System — Visual Backtest to Automated Execution

---

### **1. Objective**

Develop a Binance Perpetual Futures trading system that:

1. **Is not indicator-based** — relies on price action & market structure.
2. Can be **visually backtested** in TradingView.
3. Can **execute trades live** on Binance via webhooks.
4. Can later be **fully migrated** to a standalone Binance API bot.

---

### **2. Development Phases**

#### **Phase 1 — TradingView Pine Script**

* **Purpose:** Build & refine strategy logic in a visual environment.
* **Deliverables:**

  * Pine Script strategy with adjustable parameters.
  * Uses **price action rules** (e.g., breakouts, liquidity grabs, supply-demand zones).
  * Plots entries, exits, and PnL on chart.
  * Full backtest capability on historical data in TradingView.
* **Outcome:** Finalized and optimized strategy with proven backtest metrics.

#### **Phase 2 — Webhook Execution**

* **Purpose:** Send TradingView alerts to Binance for live trading.
* **Deliverables:**

  * Pine Script alert conditions matching strategy logic.
  * JSON-formatted alert messages.
  * A middleware service (e.g., via TradingConnector, custom Python Flask app, or 3rd-party tool) to receive alerts and send orders to Binance API.
* **Outcome:** Live automated trading via TradingView → Webhook → Binance.

#### **Phase 3 — Full Binance API Bot**

* **Purpose:** Remove dependency on TradingView for execution.
* **Deliverables:**

  * Python-based bot using Binance Futures API.
  * Same core logic from Pine Script implemented in Python.
  * Backtesting module using Binance historical data (via Backtrader or Vectorbt).
  * Optional GUI or CLI for managing bot parameters.
* **Outcome:** Fully independent, 24/7 running Binance trading bot.

---

### **3. Functional Requirements**

#### **Strategy Rules (Example Draft)**

* **Trade Type:** Long/Short allowed.
* **Entry Logic:**

  * Price breaks above/below recent key swing level.
  * Confirmation via volume spike or candle structure.
* **Exit Logic:**

  * Take profit at fixed R\:R ratio or trailing stop.
  * Stop loss placed just beyond invalidation level.
* **Risk Management:**

  * Position sizing based on % of account equity.
  * Daily loss limit.

#### **Backtest Requirements**

* Must display:

  * Win rate
  * Average RR
  * Max drawdown
  * Net profit %
* Must visually mark trades on chart.

#### **Webhook Execution Requirements**

* Alert format must include:

  ```json
  {
    "side": "BUY",
    "symbol": "BTCUSDT",
    "type": "MARKET",
    "quantity": "0.01"
  }
  ```
* Middleware must:

  * Receive HTTP POST from TradingView.
  * Validate request & security key.
  * Place order via Binance API.

#### **Full Bot Requirements**

* Run 24/7 on VPS or local server.
* Adjustable parameters via config file.
* Logging & error handling.
* Optional Telegram notifications for trade activity.
