### **1. Project Overview**

This project aims to create a **profitable, non-indicator-based trading strategy** for Binance perpetual futures, with an initial deployment on **TradingView** for visual backtesting, followed by webhook-based live execution to Binance, and eventual migration to a fully autonomous Binance API trading bot.

---

### **2. Goals & Objectives**

1. **Backtest visually on TradingView** using Pine Script for strategy validation.
2. **Execute live trades on Binance Futures** via TradingView alerts + webhooks.
3. **Transition to a Binance-native API bot** for 24/7 trading without TradingView dependency.
4. Strategy must be **non-indicator based**, relying on **price action & order book concepts**.

---

### **3. Development Phases**

#### **Phase 1: TradingView Strategy Development**

* **Platform:** TradingView (Pine Script v5)
* **Core Strategy Logic:**

  * No lagging indicators (e.g., no RSI, MACD, EMA).
  * Use pure price action logic: market structure, swing highs/lows, volatility thresholds, time-based entries, or volume from Binance data feed if available.
  * Risk management: fixed % per trade, stop-loss & take-profit levels.
* **Backtest Requirements:**

  * Display entry/exit points on chart.
  * Show profit/loss, win rate, drawdown.
  * Allow adjustable parameters for optimization.

---

#### **Phase 2: Webhook Execution to Binance**

* **Method:** TradingView alerts → Webhook → Binance API order execution service.
* **Execution Middleware:**

  * Small server app (Python/Node.js) to receive alerts and send authenticated requests to Binance Futures API.
  * Support for both LONG and SHORT trades, order sizing, and stop-loss/take-profit placement.
* **Security:**

  * Use Binance API key/secret securely stored (not in code).
  * IP whitelisting on Binance API settings.

---

#### **Phase 3: Binance API Native Bot**

* **Language:** Python preferred for ease of development & integration with backtesting libraries.
* **Features:**

  * Fetch live market data from Binance WebSocket stream.
  * Apply the same strategy logic as Pine Script version.
  * Place and manage orders directly on Binance Futures.
  * Include logging, error handling, and restart resilience.
  * Optional: Telegram bot integration for trade notifications.

---

### **4. Deliverables**

1. **TradingView Pine Script strategy** (.pine file) with full inline documentation.
2. **Webhook middleware** (Python/Node.js) for Binance order execution.
3. **Binance API bot** (Python) for direct trading without TradingView.
4. **User Manual** explaining setup, usage, parameters, and risk controls.
5. **Test & Backtest Reports** (win rate, profit factor, max drawdown).

---

### **5. Risks & Contingencies**

* **Risk:** Binance API changes → **Mitigation:** Use official API docs & update regularly.
* **Risk:** TradingView Pine Script execution limits → **Mitigation:** Keep logic optimized.
* **Risk:** Webhook delays/slippage → **Mitigation:** Use low-latency server close to Binance servers.
* **Risk:** Strategy drawdown → **Mitigation:** Strict risk management & forward testing before scaling.
