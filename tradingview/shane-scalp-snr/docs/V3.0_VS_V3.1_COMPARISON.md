## 🐛 **Bug #1: Stop Loss Execution**

### **v3.0 Code (BROKEN)** ❌
```pinescript
// Line 792 - Creates STOP ORDER, not stop loss!
if buy_signal and enable_strategy and not max_trades_reached and strategy.position_size == 0
    // ... TP calculation ...
    entry_price := close
    stop_loss := suggested_sl
    trades_today += 1
    alert_msg = "..."
    strategy.entry("Long", strategy.long, stop=suggested_sl, alert_message=alert_msg)
    // ❌ NO STOP LOSS PROTECTION! Position will run indefinitely
```

**What Actually Happens:**
- `stop=suggested_sl` tells Pine: "Don't enter LONG until price reaches suggested_sl"
- This creates a **buy-stop order** (enters when price goes UP to sl level)
- For a LONG entry at current price, this makes no sense
- **Result: Position has NO stop loss at all** 💀

**Backtest Impact:**
- Losing trades never close automatically
- Drawdown unlimited
- Account can blow on a single bad trade

---

### **v3.1 Code (FIXED)** ✅
```pinescript
// Line 886-891 - Proper stop loss via strategy.exit()
if buy_signal and enable_strategy and not max_trades_reached and strategy.position_size == 0
    // ... TP calculation ...
    entry_price := close
    stop_loss := suggested_sl
    trades_today += 1
    alert_msg = "..."
    
    // ✅ Enter at market (no stop parameter)
    strategy.entry("Long", strategy.long, alert_message=alert_msg)
    original_position_size := strategy.position_size
    
    // ✅ Separate exit order for stop loss
    if use_trailing_stop
        risk_dist = math.abs(entry_price - stop_loss)
        trail_trigger = entry_price + (risk_dist * trail_activation)
        trail_offset_points = risk_dist * trail_offset
        strategy.exit("Long SL/Trail", "Long", stop=stop_loss, 
                      trail_price=trail_trigger, trail_offset=trail_offset_points)
    else
        strategy.exit("Long SL", "Long", stop=stop_loss)
    // ✅ Stop loss now protects the position!
```

**What Happens Now:**
1. `strategy.entry()` enters at market price immediately
2. `strategy.exit()` creates protective stop loss order
3. If price hits stop_loss, position closes automatically
4. Risk is limited to planned amount

**Verification Steps:**
```
1. Add v3.1 to chart
2. Let it generate a signal and enter
3. Check Strategy Tester > List of Trades
4. You should see "Exit: Long SL" when SL hit
5. Position should close at exact SL price (±slippage)
```

## 🐛 **Bug #2: Take Profit Sizing**

### **v3.0 Code (BROKEN)** ❌
```pinescript
// Lines 819-826 - Uses qty_percent (percentage of REMAINING)
if strategy.position_size != 0
    if is_long_position and strategy.position_size > 0
        if use_multiple_tp
            // ❌ TP1: Close 50% of current position
            if not tp1_hit and high >= tp1_price
                strategy.close("Long", qty_percent=tp1_percent, comment="TP1", ...)
                tp1_hit := true
            
            // ❌ TP2: Close 30% of REMAINING (not original!)
            if tp1_hit and not tp2_hit and high >= tp2_price
                strategy.close("Long", qty_percent=tp2_percent, comment="TP2", ...)
                tp2_hit := true
            
            // TP3: Close whatever is left
            if tp2_hit and high >= tp3_price
                strategy.close("Long", comment="TP3", ...)
```

**Math Breakdown (1.0 lot entry):**
| Step | Intended | v3.0 Actual | Error |
|------|----------|-------------|-------|
| **Entry** | 1.0 lot | 1.0 lot | ✅ |
| **TP1 (50%)** | Close 0.5, remain 0.5 | Close 0.5, remain 0.5 | ✅ |
| **TP2 (30%)** | Close 0.3, remain 0.2 | Close 0.15 (30% of 0.5), remain 0.35 | ❌ |
| **TP3** | Close 0.2 | Close 0.35 | ❌ |

**Why It's Wrong:**
- `qty_percent` is **relative to current position size**
- After TP1 closes 50%, only 0.5 lot remains
- TP2's `qty_percent=30` means "30% of 0.5 = 0.15"
- You're holding MORE risk at TP2/TP3 than intended

---

### **v3.1 Code (FIXED)** ✅
```pinescript
// Lines 938-970 - Uses qty (absolute quantity from original)
var float original_position_size = 0.0  // ✅ Track original size

// At entry
if buy_signal and enable_strategy ...
    strategy.entry("Long", strategy.long, ...)
    original_position_size := strategy.position_size  // ✅ Save original

// TP management
if strategy.position_size != 0
    if is_long_position and strategy.position_size > 0
        if use_multiple_tp
            // ✅ TP1: Close 50% of ORIGINAL
            if not tp1_hit and high >= tp1_price
                close_qty = original_position_size * (tp1_percent / 100.0)  // = 0.5
                strategy.close("Long", qty=close_qty, comment="TP1", ...)
                tp1_hit := true
            
            // ✅ TP2: Close 30% of ORIGINAL
            if tp1_hit and not tp2_hit and high >= tp2_price
                close_qty = original_position_size * (tp2_percent / 100.0)  // = 0.3
                strategy.close("Long", qty=close_qty, comment="TP2", ...)
                tp2_hit := true
            
            // ✅ TP3: Close remaining (20% of original)
            if tp2_hit and high >= tp3_price
                strategy.close("Long", comment="TP3", ...)  // Closes all remaining
```

**Math Breakdown (1.0 lot entry):**

| **TP1 (50%)** | Close 0.5, remain 0.5 | Close 0.5, remain 0.5 | ✅ |
| **TP2 (30%)** | Close 0.3, remain 0.2 | Close 0.3, remain 0.2 | ✅ |

### **v3.0 Code (BROKEN)** ❌
```pinescript

    _conf += touches >= 3 ? 1 : 0  // Strong level bonus

        _conf += cisd_confidence_boost

```

**Problem:**
- Every signal gets +2 confidence regardless of trend direction
- Counter-trend signals look as good as trend-aligned signals
- Confidence scores become meaningless:
  - Counter-trend sweep: 3 + 0 + 1 + 1 = **5** (appears "high confidence")
  - Trend-aligned sweep: 3 + 1 + 1 + 1 = **6** (only +1 difference!)
  
**Real Example:**
```

Signal: BUY sweep at support (counter-trend)

v3.0 Confidence Calculation:
- Base: 3
- Strong level: +1 (3+ touches)
- M5 trend: +1 (❌ even though M5 is bearish!)
- H1 trend: +1 (❌ even though H1 is bearish!)
- Total: 6 (shown as "🔥 HIGH CONFIDENCE")

Result: Counter-trend trade looks high quality when it's not!
```

## 🐛 **Bug #4: TP Lines Memory Leak**

### **v3.0 Code (BROKEN)** ❌
```pinescript
// Lines 873-886 - Creates new lines EVERY bar
if show_tp_lines and strategy.position_size != 0 and barstate.islast
    // ❌ New line created - old one NOT deleted
    line.new(bar_index - 10, tp1_price, bar_index + 10, tp1_price, 
             color=color.new(color.green, 30), width=2, ...)
    label.new(bar_index + 5, tp1_price, "TP1", ...)
    
    if use_multiple_tp
        // ❌ More lines created every bar
        line.new(bar_index - 10, tp2_price, ...)
        label.new(bar_index + 5, tp2_price, "TP2", ...)
        line.new(bar_index - 10, tp3_price, ...)
        label.new(bar_index + 5, tp3_price, "TP3", ...)
    
    // ❌ SL line also created every bar
    line.new(bar_index - 10, stop_loss, ...)
    label.new(bar_index + 5, stop_loss, "SL", ...)
```

**What Happens:**
1. Position opens at bar 100
2. Bar 101: Creates 4 lines + 4 labels (TP1/TP2/TP3/SL)
3. Bar 102: Creates 4 MORE lines + 4 MORE labels (8 + 8 total)
4. Bar 103: Creates 4 MORE lines + 4 MORE labels (12 + 12 total)
5. ... continues until Pine's limit (100 lines) is hit
6. After limit: No new drawings render, chart becomes messy

**Visual:**
```
Bar 100: Entry
Bar 101: ║ ║ ║ ║  (4 lines)
Bar 102: ║║ ║║ ║║ ║║  (8 lines, overlapping)
Bar 103: ║║║ ║║║ ║║║ ║║║  (12 lines, stacked)
Bar 150: ███████████████  (200 lines, chart unusable)
```

---

### **v3.1 Code (FIXED)** ✅
```pinescript
// Lines 975-978 - Persistent line references
var line tp1_line = na
var line tp2_line = na
var line tp3_line = na
var line sl_line = na
var label tp1_label = na
var label tp2_label = na
var label tp3_label = na
var label sl_label = na

// Lines 980-1006 - Cleanup before creating new
if barstate.islast
    // ✅ Delete old lines first
    if not na(tp1_line)
        line.delete(tp1_line)
        line.delete(tp2_line)
        line.delete(tp3_line)
        line.delete(sl_line)
        label.delete(tp1_label)
        label.delete(tp2_label)
        label.delete(tp3_label)
        label.delete(sl_label)
    
    // Reset references
    tp1_line := na
    tp2_line := na
    tp3_line := na
    sl_line := na
    tp1_label := na
    tp2_label := na
    tp3_label := na
    sl_label := na
    
    // ✅ Create new ones only if position exists
    if show_tp_lines and strategy.position_size != 0
        tp1_line := line.new(...)
        tp1_label := label.new(...)
        if use_multiple_tp
            tp2_line := line.new(...)
            tp2_label := label.new(...)
            tp3_line := line.new(...)
            tp3_label := label.new(...)
        sl_line := line.new(...)
        sl_label := label.new(...)
```

**What Happens Now:**
1. Position opens at bar 100
2. Bar 101: Creates 4 lines + 4 labels, saves references
3. Bar 102: Deletes bar 101's lines, creates new 4 lines + 4 labels
4. Bar 103: Deletes bar 102's lines, creates new 4 lines + 4 labels
5. ... always maintains exactly 4 lines + 4 labels
6. Position closes: Deletes all lines, sets references to na

**Visual:**
```
Bar 100: Entry
Bar 101: ║ ║ ║ ║  (4 lines)
Bar 102: ║ ║ ║ ║  (4 lines, old deleted)
Bar 103: ║ ║ ║ ║  (4 lines, old deleted)
Bar 150: ║ ║ ║ ║  (still only 4 lines, clean!)
```

**Verification:**
```

3. Right-click chart > Objects Tree
```
